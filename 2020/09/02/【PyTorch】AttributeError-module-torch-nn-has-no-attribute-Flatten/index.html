【PyTorch】AttributeError: module 'torch.nn' has no attribute 'Flatten'

2020-09-02
Word count: 555   |   Reading time≈ 2 min

问题简述

​ 当数据集为图像数据，在利用nn.Sequential类构建神经网络时，往往需要先利用nn.Flatten(start_dim=1)将数据展平 —— 使数据集变为矩阵张量（行长度为batch_size，列长度为特征量），但在导入pytorch-cpu库运行以下代码时却遇到异常：AttributeError: module ‘torch.nn’ has no attribute ‘Flatten’。

运行环境

  • pytorch-cpu：1.1.0

    若是使用GPU版，则大概率可以通过升级PyTorch解决此问题

    • pip命令：pip install –upgrade pytorch==1.2.0
    • conda命令：conda update pytorch==1.2.0

  • CUDA：None

解决思路

  • Step1. 在PyTorch的Github项目（https://github.com/pytorch/pytorch）中查询`nn.Flatten`

    • 发现：Flatten类的对应源码为 nn/Modules/flatten.py
    • 说明：目前PyTorch的版本可能已经进行了相应的更新，可以尝试更新
    • 命令：conda update pytorch-cpu
    • 效果：torch版本依然为1.1.0，且代码依旧报错

  • Step2. 查看nn.Module下其他类是否能够正常运行

    • 操作：在VS code中输入nn.，看代码提示中是否出现了Linear或BatchNorm2d

    • 若正常显示，表示nn.Module类可以正常导入并调用，则Flatten类不能调用的原因是nn.Modules声明包（__init__.py)时没有从flatten.py导入Flatten类，导致在调用nn.Modules包时无法找到Flatten类

  • Step3. 修改nn.Modules的包声明文件

    • 修改前，可以看到包声明文件并没有导入Flatten类

    • 在导入部分导入Flatten类

    • 在__all__数组的尾部加入类名，注意以逗号结尾

效果

总结

  • 问题产生原因

    pytorch-cpu的1.1.0版本，其nn.Modules包中有flatten.py文件（定义了Flatten类），但并没有在Modules的包声明文件中导入该类，以致于在调用nn.Modules使无法找到该类。

  • 解决办法

    在Modules的包声明文件导入该类，并在__all__数组中加入该类名

  • 补充说明

    截止2020.09.02，PyTorch还在flatten.py中定义了Unflatten类，而pytorch-cpu的1.1.0版本亦未对此类进行声明，因此也可以依据上述办法解决AttributeError: module ‘torch.nn’ has no attribute ‘Unflatten’ 异常。
